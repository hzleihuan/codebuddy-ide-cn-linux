# 4.9.15 → 4.10.0 版本变更记录

上游包：`CodeBuddy-linux-x64-4.10.0.32999201-c8bdde62-cn.deb`

> **重大变化：上游来源从 macOS DMG 切换为官方 Linux x64 .deb。**
> 从 4.10.0 起，官方已提供 Linux x64 `.deb` 安装包，本项目整体从「解包 macOS DMG → 转 loong64」迁移为「解包官方 Linux x64 .deb → 转 loong64」。因此不再需要 ICNS 图标转换、Info.plist 读取、Mach-O 清理以及 Windows/macOS 专有模块剥离等 macOS 相关逻辑；Electron 版本改为从 `.deb` 内 `package.json` 的 `devDependencies` 自动读取。

## Electron 版本

| 版本 | 4.9.15 | 4.10.0 | 变化 |
|---|---|---|---|
| Electron | 35.6.0 | **37.7.0** | 大版本升级 |
| App Version | 1.102.3 | **1.106.1** | 升级 |

Electron 从 35 大版本升级到了 37 大版本，Node.js ABI 发生变化，所有原生模块须针对新版本重新编译。

本项目通过 `detect_electron_version()` 自动从 `.deb` 内 `resources/app/package.json` 的 `devDependencies.electron` 读取实际 Electron 版本（本版本为 `37.7.0`），因此**运行时下载和原生模块编译无须手动指定**。同步更新 `install.sh` 中的 fallback 默认值和 `README.md` 中的 Electron 徽章。

> 说明：4.9.15 的 App Version 取自 macOS DMG 的 `CFBundleShortVersionString`；4.10.0 的 App Version 取自 Linux .deb 内 `package.json` 的 `version` 字段。

## 模块变化分析与移植处理

### 1. `@tencent/qimei-node` 与 `koffi` 在 Linux .deb 中不再存在

- **背景**：4.9.15 的分析对象是 macOS DMG，其中包含腾讯私有模块 `@tencent/qimei-node`（QIMEI 设备可信标识）及其 Windows 平台 FFI 依赖 `koffi`。这两个模块在 `package.json` 中均为 `optionalDependencies`：
  ```json
  "optionalDependencies": {
      "@tencent/qimei-node": "~1.2.2",
      "windows-foreground-love": "0.5.0"
  }
  ```
- **现状**：官方 Linux x64 `.deb` 内的 `node_modules` **根本不包含** `@tencent/qimei-node` 与 `koffi` 目录。这是因为该模块仅实现了 Windows（`src/win/`）与 macOS（`src/mac/`）底层能力，在 Linux 平台安装时作为可选依赖被跳过，与其主进程逻辑一致：
  ```javascript
  if (process.platform !== "darwin" && process.platform !== "win32") {
      this.available = false;
      return;
  }
  ```
- **移植处理**：4.9.15 中在 `remove_known_wrong_platform_modules()` 新增的 qimei-node / koffi 清理逻辑对 Linux .deb 而言已成为空操作（目录不存在时 `rm -rf ... || true` 不产生副作用），予以保留即可，**无需新增处理**，也不影响构建。

### 2. `node-pty@1.1.0-beta33 → 1.1.0-beta35`

- **类型**：关键原生模块（终端 PTY 支持），位于 `critical_modules` 列表。
- **移植处理**：由 `rebuild_native_modules()` 按 `package.json` 中的实际版本自动从 npm 拉取对应源码，并针对新的 loong64 Electron 37 头文件重新编译，**无需修改脚本**。

### 3. `@parcel/watcher@2.5.1 → 2.5.6`

- **类型**：文件系统监听平台模块，由 `refresh_platform_packages()` 从 npm 拉取 loong64 适配版本。
- **移植处理**：`refresh_platform_packages()` 已按 `package.json` 声明版本自动获取最新对应版本，**无需修改脚本**。

## 代码变更

本版本主体变更为「DMG → Linux x64 .deb」的管线迁移（见 commit `e7fe2f1`），版本适配层面的增量改动如下。

### `install.sh`

```diff
-ELECTRON_VERSION="${ELECTRON_VERSION:-35.6.0}"
+ELECTRON_VERSION="${ELECTRON_VERSION:-37.7.0}"
```

> Electron 版本仍以 `detect_electron_version()` 从 `package.json` 自动检测为准，此处仅同步更新检测失败时的 fallback 默认值。

### `README.md`

```diff
-<img src="...Electron-v35.6.0..." alt="Electron Version">
+<img src="...Electron-v37.7.0..." alt="Electron Version">
```

### `scripts/lib/native-modules.sh`

无需新增处理。4.9.15 为 macOS DMG 引入的 `@tencent/qimei-node`、`koffi` 清理逻辑对 Linux .deb 为空操作，保留即可；其余原生与平台模块由现有 `critical_modules` / `optional_modules` / `refresh_platform_packages()` 机制按 `package.json` 声明版本自动处理。
