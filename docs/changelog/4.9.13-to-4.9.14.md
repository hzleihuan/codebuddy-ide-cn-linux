# 4.9.13 → 4.9.14 版本变更记录

上游 DMG：`CodeBuddy-darwin-x64-4.9.14.31414011-6230de8a-cn.dmg`

## Electron 版本

| 版本 | 4.9.13 | 4.9.14 |
|---|---|---|
| Electron | 34.5.1 | **35.6.0** |
| App Version (CFBundleShortVersionString) | 1.100.0 | **1.102.3** |

Electron 从 34 大版本升级到了 35 大版本，Node.js ABI 可能变化，所有原生模块须针对新版本重新编译。

项目通过 `detect_electron_version()` 自动从 DMG 的 `Info.plist` 读取实际 Electron 版本，因此**运行时下载和原生模块编译无须手动指定**。本次同步更新了 `install.sh` 中的 fallback 默认值和 `README.md` 中的 Electron 徽章。

## 新增移植处理的模块

以下三个模块在 4.9.13 中就已存在，但之前项目未做针对性处理。本次一并补齐。

### native-is-elevated@0.7.0

- **类型**：原生模块（有 `binding.gyp` + `src/iselevated.cc`）
- **npm 源码**：完整，可从 npm 获取并编译
- **Linux 兼容性**：完全兼容，C++ 源码使用 `#ifdef` 区分平台，Linux 分支调用 `geteuid() == 0`
- **实际行为**：`index.js` 入口在非 Windows 平台直接返回 `process.getuid() === 0`，**不加载 `.node` 文件**，因此 Linux 上即使不编译也不影响功能
- **移植处理**：加入 `optional_modules` 列表，从 npm 拉取源码重新编译。编译失败不阻塞构建

### @vscode/policy-watcher@1.3.2

- **类型**：原生模块（有 `binding.gyp` + 多平台 C++ 源码，依赖 `bindings` + `node-addon-api`）
- **npm 源码**：完整，包含 `src/linux/PolicyWatcher.cc`（空实现 stub）
- **Linux 兼容性**：编译可以成功。Linux 的 C++ 实现全部是空方法；`main.cc` 在非 Windows/macOS 平台 `createWatcher()` 会 `throw TypeError("Unsupported platform")`
- **实际行为**：`index.js` 通过 `require('bindings')('vscode-policy-watcher')` **无条件加载 `.node` 文件**，因此 Linux 上必须编译出 `.node` 才能避免加载报错。上游 VS Code 在调用处有 `try-catch` 包裹，运行时抛异常不影响正常使用
- **移植处理**：加入 `optional_modules` 列表，从 npm 拉取源码重新编译。编译失败不阻塞构建

### @vscode/deviceid@0.1.1

- **类型**：纯 JS 模块（`main` 为 `./dist/index.js`，依赖 `fs-extra` + `uuid`）
- **Linux 兼容性**：完全兼容，无需编译
- **残留文件**：DMG 中包含 `build/Release/windows.node`，仅 Windows 平台使用
- **移植处理**：在 `remove_known_wrong_platform_modules()` 中新增清理 `windows.node` 的逻辑
